# tools — 기출 PDF → 앱 데이터 파이프라인

`app/data/exams.json`을 원본 PDF에서 다시 만들어 내는 스크립트다.
새 회차 기출이 추가되면 이 순서대로 돌린다. **저장소 루트에서 실행한다.**

```bash
export PYTHONPATH=tools          # joiner/lexicon을 서로 import 한다

python tools/build_lex.py  lex.json                          # 0. 어절·음절쌍 통계
python tools/extract.py    blocks.json lex.json              # 1. 표 / 박스 지문 추출
python tools/merge.py      blocks.json 기존.json merged.json  # 2. 지문에서 표·박스 분리
python tools/respace.py    merged.json final.json lex.json   # 3. 띄어쓰기 복원

cp final.json app/data/exams.json
node tools/smoke_test.js app                                 # 4. 렌더러 검증
```

마지막에 `app/sw.js`의 `CACHE` 값을 올려야 기기에 캐시된 옛 데이터가 갱신된다.
`app/` 수정분은 `docs/`(GitHub Pages)에도 복사한다.

## 왜 이렇게까지 하나

원본 PDF에는 두 가지 함정이 있다.

**표와 박스가 좌표로만 존재한다.** 텍스트로 뽑으면 `비목 구분 운송비 보관비 6,000 2,000 …`
처럼 한 줄로 납작해진다. 그래서 괘선(선분)을 연결요소로 묶어 테두리 영역을 찾고,
그 안의 세로·가로 괘선으로 격자를 세운 뒤 셀마다 글자를 중심좌표로 담는다.

**공백 글자가 없다.** 단어 사이를 글자 위치로만 벌려 놓아서 `화물자동차운수사업은`처럼
붙어 나온다. 글자 간격이 폰트 크기의 30%를 넘으면 띄어쓰기로 본다
(단어 사이 0.5~0.7배, 글자 사이 0.1배 미만으로 뚜렷이 갈린다).

## 파일

| 파일 | 하는 일 |
|---|---|
| `lexicon.py` | PDF의 각 시각적 줄을 띄어쓰기 복원된 문자열로 뽑는다 |
| `joiner.py` | 줄바꿈 지점에서 띄어쓸지 붙일지 판정한다 |
| `build_lex.py` | 위 둘로 어절·음절쌍 빈도표(`lex.json`)를 만든다 |
| `extract.py` | 표·박스를 좌표에서 복원한다 |
| `merge.py` | 복원한 블록을 `exams.json`에 붙이고 지문의 중복 텍스트를 지운다 |
| `respace.py` | 지문·선지의 띄어쓰기를 복원한다 |
| `smoke_test.js` | 브라우저 없이 `app/js/app.js`의 렌더러를 구동해 검증한다 |
| `make_icons.py` | PWA 아이콘(192/512/apple-touch)을 생성한다 |

## 까다로운 지점들

- **수송표**는 셀 안에 운임 표시용 소형 칸이 또 있어 격자가 잘게 쪼개진다.
  대각선 분할 셀이 있으면 표 전체를 가로지르는 주요 괘선만 쓴다.
- **괘선이 셀 단위로 끊겨** 그려진 표가 많다. 같은 직선 위 선분을 먼저 이어 붙여야
  '행을 가로지르는 선'을 알아볼 수 있고, 그래야 병합 셀 판정이 맞는다.
- **줄바꿈**은 시험지가 양쪽 정렬이라 단어 한가운데서 끊긴다(`이산화탄` / `소배출계수`).
  좌표만으로는 알 수 없어, 줄 안쪽에서 얻은 어절·음절쌍 통계의 로그오즈로 판정한다.
  leave-one-out 검증 기준 어절 경계 95.9% / 어절 중간 95.7%. 나머지 약 4%는
  `자료 에서`처럼 어색한 공백이 남는다.
- **과목명 배너**(`화 물 운 송 론`)도 테두리 박스라 걸린다. 이름으로 걸러낸다.

## 안전장치

`merge.py`와 `respace.py`는 **공백을 뺀 글자열이 원본과 완전히 같을 때만** 값을 바꾼다.
글자가 하나라도 달라지면 기존 값을 그대로 둔다. 정답·해설은 아예 건드리지 않는다.
